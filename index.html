Executive Summary

A geosciences data
infrastructure platform

LithoQuest is an AI-powered geosciences data infrastructure platform built to resolve a critical bottleneck in the physical sciences: the trapped value of unstructured fieldwork data. By utilizing a Retrieval-Augmented Generation (RAG) architecture, LithoQuest ingests dense, unstructured geological and geophysical reports and transforms them into highly structured, queryable, and spatially mapped JSON arrays.

The platform operates not as a conversational chatbot, but as a rigid, programmatic reasoning engine. It reads field reports, identifies critical geochemical and geophysical parameters, and outputs clean data ready for immediate database insertion — with LLMs serving strictly as internal extraction engines, severely constrained by technical system prompts to eliminate hallucination.

Currently operating in its Functional Beta phase (Quest MVP), LithoQuest is being developed and rigorously tested within a closed, academic environment by peer geosciences researchers at the University of Nigeria, Nsukka (UNN). The core platform leverages Amazon Bedrock (with localized open-source fallback models) as its extraction backbone, with structured outputs routed to MongoDB Atlas via a serverless, auto-scaling infrastructure.

By automating the tedious data-structuring phase of geological exploration, LithoQuest empowers geoscientists to focus on high-level spatial mapping and resource evaluation — modernizing the mining and geosciences sector with a scalable, secure, and highly accurate computational foundation for sustainable mineral and groundwater exploration across West Africa and beyond.

The Industry Problem

Exploration is stuck in manual mode

Geological exploration, groundwater management, and mineral processing rely heavily on historical and active field reports. These reports are almost exclusively unstructured text documents and PDFs.

Extracting quantitative metrics — such as layer depths from Vertical Electrical Sounding (VES) surveys, or crushing size requirements for mineral liberation — is currently a slow, manual, and error-prone process. This unstructured data cannot be easily fed into modern predictive models, geographic information systems (GIS), or automated compliance pipelines.

Geology students graduate without ever touching the AI tools the modern industry demands. The gap between fieldwork and actionable intelligence has never been wider. LithoQuest closes it.

Unstructured Data Bottleneck

Field reports — PDFs, handwritten notes, assay tables — cannot be ingested directly into predictive models or GIS pipelines. Every data point requires manual extraction, introducing delay and human error.

Fragmented Geophysical Insights

VES resistivity readings, lithology descriptions, and aquifer classifications sit in disconnected documents. Without structured extraction, cross-referencing field data across surveys is prohibitively expensive.

The Compliance & Automation Gap

Automated compliance pipelines require structured, machine-readable inputs. The absence of systematic geochemical data structuring inflates time-to-discovery and raises operational costs across every survey programme.

The LithoQuest Solution

Three precise extraction capabilities

LithoQuest reads unstructured geological reports, identifies critical parameters, and outputs clean, queryable JSON — ready for immediate database insertion and spatial analysis.

Geophysical Data Extraction
VES · Resistivity · Lithology

Parsing raw text field reports to identify and structure Vertical Electrical Sounding (VES) metrics with full parameter fidelity. Extracted fields include layer_depth_m, resistivity_ohm_m, and domain-specific lithology_description parameters.

The semantic chunking engine preserves geological context throughout extraction — keeping resistivity metrics mathematically tied to their respective layer depths, preventing the data-association errors common in naive parsing approaches.

Hydrogeological Mapping
Aquifer · Water Table · GIS

Automating the detection and categorization of saturated aquifer zones and water table depths from qualitative field notes. The system converts ambiguous, prose-form hydrogeological observations into structured, machine-readable classifications — enabling direct ingestion into GIS pipelines and groundwater management systems.

This bridges the critical gap between field geologist notation conventions and the structured inputs demanded by modern spatial analysis platforms.

Mineral Processing Analysis
Liberation · Crushing · Assay

Extracting specific mineral liberation requirements from bulk assay text — including optimal crushing sizes for lepidolite and construction-grade materials. The system identifies actionable parameters buried within dense laboratory and field report prose, structuring them for direct use in mineral processing workflow optimization.

Critical battery metal exploration — particularly lithium from lepidolite — depends on precise crushing specifications that historically required manual specialist extraction from each report.

Technical Architecture

The RAG Pipeline — three phases

A decoupled, microservices approach ensures rapid processing, cost-efficiency, and minimal vendor lock-in. Every layer is purpose-built for Earth science data demands — from raw field PDFs to terabyte-scale stratigraphic archives.

Serverless scaling via MongoDB Startup program ($500 credit tier) ensures costs scale to zero during idle periods between bursty academic testing cycles.

01
Data Ingestion & Pre-processing
Python Backend
Document Parsing
A Python-driven ingestion layer strips formatting, tables, and noise from uploaded PDF field reports — producing clean prose ready for extraction.
Semantic Chunking
Cleaned text is algorithmically divided into overlapping logical chunks to preserve geological context, keeping resistivity metrics mathematically tied to their respective layer depths.

The Reasoning Engine
LLM Layer — Amazon Bedrock
Compute Allocation
Lightweight AWS EC2 instances (scaling from 8 vCPUs) handle API routing and backend logic — balancing throughput with cost during academic testing cycles.
Extraction Models
Amazon Bedrock (or localized open-source fallback) used strictly as data-extraction engines via rigid system prompts — bypassing conversational output to generate pure JSON arrays of targeted metrics.
03
The Data Storage Layer
MongoDB Atlas
Native JSON Handling
Extracted arrays pushed directly to MongoDB Atlas — its NoSQL document structure accommodates varying geological parameters without rigid SQL schema migrations.
Unified Vector Store
Atlas Vector Search houses both raw document embeddings and structured outputs in a single, unified environment — reducing latency and infrastructure complexity.
Serverless Scaling
Deployed on a Serverless Instance framework via MongoDB Startup program, ensuring infrastructure costs scale down to zero between bursty testing periods.
Development Roadmap

Functional Beta — Quest MVP

Three milestones chart the path from core architecture through live deployment at UNN's Department of Geology.

Complete
Milestone 1 — Core Architecture Defined
Core RAG architecture fully defined. Python extraction scripts written and validated. Database schemas outlined for VES and lithology data. The foundational pipeline from PDF ingestion through JSON output is structurally complete.
Active
Milestone 2 — Cloud Provisioning & Local Validation
Overcoming enterprise cloud provisioning bottlenecks, including AWS Trust & Safety clearance for the Bedrock extraction layer. Concurrently executing local parallel testing with dummy JSON generation to validate end-to-end database routing. This phase proves the pipeline before live data ingestion begins.
Upcoming
Milestone 3 — Live Deployment & Historical Data Ingestion
Full deployment of the Bedrock extraction layer in production. Ingestion of live historical geological data from the Department of Geology, University of Nigeria, Nsukka — the first real-world dataset to flow through the complete pipeline. Expansion of the Vector Search UI to support researcher queries across the accumulated stratigraphic record.
